B vs Strong

Mientras que visualmente son iguales, ya que ambas colocan el texto en negritas. Semanticamente, b, unicamente tiene efecto visual no aporta significado adicional. Por su parte, strong, si comunica la importancia fuerte al texto.

En motores, como Google, entienden strong como enfasis y, b, lo ignoran como un estilo.

Ejemplo:

Este texto fue escrito con b
Este texto fue escrito con strong

I vs Em

Ambas van a mostrar el texto en cursivas, no obstante, la primera tambien solo tendra un efecto visual(otro idioma, textos cientificos, etc). En cambio, em indica el enfasis de una palabra dentro de una oracion, incluso, se pueden anidarse

Los motores de busqueda y lectores de pantalla suelen marcar el enfasis de em, pero no el de i

Ejemplo:

Este texto ha sido escrito usando i
Ahora, aqui usamos em
S vs Del

Ambas muestran texto tachado. s, indica que el contenido ya no es válido o vigente, pero sigue visible. del, por su parte, indica que el contenido ha sido eliminado formalmente del documento. Incluso puede llevar atributos de fecha (datetime) y referencia (cite).

Los lectores de contenido suelen anunciar un texto dentro de del fue eliminado. Pero no lo hacen cuando se usa s

Ejemplo:

Este texto ha sido escrito con s
Para este texto se uso del
Elementos de linea y elementos de bloque
De Linea

Los elementos de linea se colocan dentro del flujo del texto, ya que no rompen la linea, y ocupan solo el ancho necesario de su contenido. Unicamente van a admitir margenes laterales

Por ejemplo:

  • SPAN
  • A
  • STRONG
  • EM
  • IMG
De Bloque

Siempre inician en una nueva linea, ocupan todo el ancho disponible. Asi tambien, permite margenes de alto y ancho. Y se usan para estructurar grandes secciones del documento.

Por ejemplo:

  • DIV
  • P
  • SECTION
  • HEADER
  • FOOTER
Formularios

INPUT

Es el campo mas versatil y su atributo principal, define el tipo de entrada

Ejemplos de type:

  • Text: entrada de texto corta
  • Password: texto oculto
  • Email: valida formato de correo
  • Number: numeros con min, max y step
  • Date: selector de fecha
  • Radio: Opcion excluyente dentro de un grupo
  • Checkbox: opcion multiple independiente

LABEL

Asocia un texto descriptivo a un campo de formulario. Mejora la accesibilidad y experiencia de usuario, ya que permite hacer click en el texto para activar el campo

OUTPUT

Muestra el resultado de un calculo o accion dentro de un formulario. Tiene atributos como for para indicar cambios relacionados.

PROGRESS

Representa visualmente el progreso de una tarea. Tiene los atributos value (progreso actual) y max (valor total). Si no se indica value, muestra un estado indeterminado

SELECT

Crea un menu desplegable de opciones. Tiene atributos como name, id, required, multiple, size, etc.

OPTION

Define cada opcion dentro del select, entre sus atributos se encuentra value, selected, disabled

TEXT - AREA

Campo para introducir texto multilinea. Algunos de sus atributos son: rows, cols, placeholder, min max - lenght, wrap, etc. Es ideal para comentarios largos o descripciones
